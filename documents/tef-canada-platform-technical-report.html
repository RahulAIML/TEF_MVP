TEF Canada Training Platform • Technical Report

Scope: Reading, Listening, Passage Analysis, and Performance Dashboard

Version: 2026.03 • Environment: Web (Next.js + FastAPI)

System Summary

  • • End•to•end TEF preparation system focused on Reading and Listening modules.
  • • AI•assisted content generation with validated question formats.
  • • Structured mock exams and single•item practice flows.
  • • Analytical dashboard for accuracy trends and weak•area detection.

Architecture Overview

  • • Frontend: Next.js 14, TailwindCSS, shadcn/ui, Chart.js.
  • • Backend: FastAPI with JWT auth and PostgreSQL persistence.
  • • AI: Gemini (content + fallback TTS), ElevenLabs (primary TTS).
  • • Static audio hosting served by FastAPI at /audio.

Key Non•Functional Requirements

  • • Consistent latency for question generation
  • • Deterministic response formats with validation
  • • Reliable audio delivery with fallback logic
  • • High availability on Render
  • • Modular, reusable UI components
  • • Clean, accessible interfaces

Functional Modules

Features & Flows

Reading Module

  • • Mock Exam: 40 questions, 60•minute timer
  • • Navigator: jump to any question
  • • Prefetch: next 5 questions generated in background
  • • Text Helper: explain selected text via /explain-text

Listening Module

  • • Mock Exam: 40 questions, 60•minute timer
  • • Practice Mode: single question with immediate explanation
  • • Audio controls: play, pause, stop, speed (0.75••1.5•)
  • • Transcript: word•level highlighting with auto•scroll
  • • Text Helper: transcript selection • explain

Passage Analysis

  • • 10•question passage quiz generator
  • • Highlighted text auto•fills helper input
  • • Supports full paragraph analysis

Performance Dashboard

  • • Line chart: accuracy trend
  • • Bar chart: score normalized to 40
  • • Pie chart: weak areas distribution

Authentication & Data

  • • JWT•based authentication
  • • Exam attempts stored in PostgreSQL
  • • Error type aggregation for weak•area analysis
  • • Session•scoped caching to avoid duplicates
  • • Full audit of question metadata
  • • Safe retries on AI generation

Technical Interfaces

APIs & Integration

Core Endpoints

  • POST /generate-question • reading question
  • POST /generate-passage-quiz • passage + 10 Qs
  • POST /generate-listening-question • script + audio
  • POST /explain-text • text explanation
  • POST /submit-exam • submit mock exam
  • GET /dashboard/summary • metrics

Audio Generation Pipeline

  • • Script generated via Gemini
  • • Primary TTS: ElevenLabs (MP3)
  • • Fallback TTS: Gemini (WAV)
  • • Audio saved to local store and served at /audio

Deployment & Configuration

  • • Backend: Render web service
  • • Frontend: Render static / web service
  • • Env: GEMINI_API_KEY, ELEVENLABS_API_KEY, ELEVENLABS_VOICE_ID
  • • Optional: GEMINI_TTS_MODEL, AUDIO_STORAGE_PATH

Quality Controls

  • • Response schema validation via Pydantic
  • • Duplicate•avoidance using content hashes
  • • Retry policy on AI generation failures
