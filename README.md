# TEF Canada Reading Training Platform

Full-stack platform for TEF Canada Reading preparation. It includes a complete mock exam, a passage analyzer with text explanations and 10 MCQ questions, and a performance dashboard. Questions are generated dynamically using Gemini.

## Features

### Module 1 - Reading Mock Exam
- 40 questions, 60-minute timer
- Questions generated on demand (per question number)
- Question navigator for jumping between questions
- Auto-submit when time expires
- Results with score, accuracy, completion time, and explanations

### Module 2 - Passage Analyzer
- Generate a fresh reading passage with 10 MCQ questions
- Select text to auto-fill the helper input
- Text Helper Tool explains selected text with Gemini

### Module 3 - Performance Dashboard
- Average reading accuracy
- Recent exam scores
- Weakest question type

### Authentication
- Demo mode enabled (no signup/login required)

## Tech Stack

- Frontend: Next.js 14, React, TypeScript, TailwindCSS
- Backend: FastAPI (Python)
- AI: Gemini API (Google Generative AI)
- Database: SQLite by default, Postgres supported

## Question Distribution (TEF Mock Exam)

Questions are generated by category based on question number:

- 1-7: Everyday-life documents
- 8-17: Incomplete sentences / gap-fill texts
- 18-22: Rapid reading of texts or graphics
- 23-32: Administrative and professional documents
- 33-40: Press articles

## Environment Variables

### Backend (`backend/.env`)

```env
# Gemini
GEMINI_API_KEY=your_gemini_key
OPENAI_API_KEY= # optional fallback
GEMINI_MODEL=gemini-2.0-flash

# App
FRONTEND_ORIGIN=http://localhost:3000
JWT_SECRET=change_me

# Database (optional)
# If not set, SQLite is used at ./tef.db

```

### Frontend (`frontend/.env.local`)

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## API Endpoints

### Mock Exam
- `POST /generate-question`
- `POST /submit-exam`

### Passage Analyzer
- `POST /generate-passage`
- `POST /generate-passage-quiz`
- `POST /word-meaning`

### Dashboard
- `GET /dashboard/summary`

## Example Requests

### Generate Question

```json
POST /generate-question
{
  "question_number": 12
}
```

Response:
```json
{
  "question_number": 12,
  "text": "...",
  "question": "...",
  "options": ["A. ...", "B. ...", "C. ...", "D. ..."],
  "correct_answer": "B",
  "explanation": "...",
  "question_type": "gap_fill"
}
```

### Submit Exam

```json
POST /submit-exam
{
  "started_at": "2026-03-12T10:00:00Z",
  "completed_at": "2026-03-12T10:58:30Z",
  "answers": { "1": "A", "2": "C" },
  "questions": [
    {
      "question_number": 1,
      "correct_answer": "A",
      "question_type": "everyday_life",
      "explanation": "..."
    }
  ]
}
```

Response:
```json
{
  "score": 28,
  "total": 40,
  "accuracy": 70.0,
  "completion_time": 3510,
  "results": [
    {
      "question_number": 1,
      "correct_answer": "A",
      "user_answer": "A",
      "is_correct": true,
      "explanation": "...",
      "question_type": "everyday_life"
    }
  ]
}
```

## Run Locally

### Backend

```bash
cd backend
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

The database tables are created automatically on startup.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Open:
```
http://localhost:3000
```

## Notes

- Mock exam and passage analyzer do not require login in demo mode.
- Questions are generated on demand to reduce latency and cost.
- SQLite is the default database for quick local testing; Postgres can be enabled via `DATABASE_URL`.
